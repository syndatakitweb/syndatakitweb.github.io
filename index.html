What it is

A statistical mirror,
not a random number generator.

syndatakit learns the full statistical structure of a dataset - marginal distributions, cross-variable correlations, temporal dynamics, volatility clustering, tail dependence - then generates unlimited new rows that are structurally identical to the original, with zero real individuals inside.

01

Preserves cointegration, stationarity, Granger causality and structural breaks. Every test that works on real data works on syndatakit output.

generate_credit_risk.py
from syndatakit.generators  import GaussianCopulaGenerator
from syndatakit.catalog     import load_seed
from syndatakit.calibration  import get_priors, apply_scenario
from syndatakit.fidelity    import fidelity_report

# Fit with Bayesian priors - stable even on small datasets
gen = GaussianCopulaGenerator(priors=get_priors("credit_risk"))
gen.fit(load_seed("credit_risk"))

# 100K high-risk records with filters
df = gen.sample(100_000, filters={
    "fico_band":   ["300-579", "580-669"],
    "dti_min":     45,
    "default_12m": 1,
})

# Apply credit crisis scenario
stressed = apply_scenario(df, "credit_crisis", intensity=0.9)

# Full fidelity report
report = fidelity_report(load_seed("credit_risk"), df)
print(f"Fidelity: {report['summary']['overall_fidelity']}%")

Privacy & compliance

Four tests.
One verdict.

Four independent attacks run automatically. Each returns a risk level. The overall verdict is the maximum risk across all tests, with a concrete recommendation.

Exact Copy Check
Zero tolerance
Hashes every synthetic row, checks against real data. Count must be 0. Always passes on properly fitted generators.
Membership Inference
AUC test
Shadow model attack. AUC near 0.5 = no advantage. AUC above 0.7 = memorisation risk detected.
Singling-Out
QI attack
Random quasi-identifier subset attack. Can an adversary uniquely identify one individual by matching on 2–4 attributes?
Linkability
NN attack
Nearest-neighbour cross-dataset attack. Can an adversary link a synthetic record back to a specific real individual? Baseline 0.5.

Core mathematics

The equations
behind the scores.

The three fidelity metrics and the copula construction. All computed at N=50,000 synthetic rows against published aggregate statistics.

01 - MARGINAL FIDELITY
Moment matching
score = 1 - (
0.40 * |μ_s - μ_r| / |μ_r|
+ 0.35 * |σ_s - σ_r| / σ_r
+ 0.15 * |γ_s - γ_r| / |γ_r|
+ 0.10 * |κ_s - κ_r| / |κ_r|
)
For kurt > 10 (Pareto variables), switches to robust estimators: median and IQR replace mean and std, and skew is dropped entirely because sample skewness is unreliable for distributions with α < 3.
02 - DISTRIBUTIONAL FIT
KS-style score
D_n = sup_x |F_synth(x) - F_ref(x)|

score = max(0, 1 - D_n * 2.5)
Two-sample Kolmogorov-Smirnov test between the synthetic data and a 100,000-point reference sample drawn from the theoretical quantile function. Measures shape agreement independently of moment accuracy.
03 - CORRELATION FIDELITY
Spearman-Frobenius
score = 1 - ||R_synth - R_real||_F
/ ||R_real||_F * 0.6
Frobenius norm distance between Spearman rank correlation matrices. Spearman (not Pearson) is used because the Gaussian copula preserves rank order by construction. Pearson is systematically attenuated for heavy-tailed distributions.
04 - COMBINED SCORE
Weighted composite
overall = 0.45 * marginal
+ 0.30 * KS
+ 0.25 * correlation
Marginal receives the highest weight because it directly measures whether individual variables match their published statistical profile. KS and correlation weight equally as distributional shape vs joint structure.
05 - GAUSSIAN COPULA
Copula construction
C(u₁,...,uₙ) = Φₙ(
Φ⁻¹(u₁),...,Φ⁻¹(uₙ); R
)

X_i = F_i⁻¹(u_i)
Correlated uniforms are generated via Cholesky decomposition of the Spearman rank correlation matrix R. Each uniform uᵢ is then transformed to the target marginal via the quantile function F⁻¹ₙ of the appropriate parametric family.
06 - DIFFERENTIAL PRIVACY
(ε, δ)-DP mechanism
M(x) = x + Lap(Δf / ε)

R_private = R + W / n
W ~ Wishart(Σ, d+1)
Laplace mechanism on marginal statistics with sensitivity Δf. Wishart mechanism on the correlation matrix. Total privacy budget ε is tracked and reported per generation call. GDPR Article 89 compatible.
Marginal distributions

The right distribution
for every variable.

Generic synthetic data tools apply the same distribution family to every variable. syndatakit selects the appropriate family based on variable type, domain constraints, and empirical moment structure.

Distribution
Used for
Key parameter
Rationale
Log-normal
Wages, loan amounts, revenues
CV-calibrated σ
Multiplicative processes. Always positive, right-skewed, captures income distributions well for moderate skew (γ < 4).
Truncated normal
Policy rates, interest rates
Lower bound lo
Rates cannot go below a floor (often 0). Truncated normal correctly models zero-lower-bound periods without zero-inflating the distribution.
Beta
Ratios, percentages, scores
Alpha, beta from moments
Bounded support [lo, hi]. Moment-matched via method of moments. Used for LTV ratios, homeownership rates, credit utilisation.
Pareto (Lomax)
Firm revenue, bank assets, transaction sizes
Median-calibrated scale
Power-law tail. Scale calibrated to median (not mean) because mean is unstable for α < 2. Robust scoring uses IQR.
Bimodal mixture
Loan terms (15yr vs 30yr)
Component means, weights
Multi-peaked distributions that no unimodal family can capture. Two normal components with calibrated mixture weights.
Regime-switching
Inflation (all profiles)
pi_high, high-regime moments
Two log-normal components: normal-regime (mean ~2.5%) and high-inflation regime (mean ~14%). Calibrated to IRS/FRED historical frequency.
Zero-inflated log-normal
FDI, sparse economic variables
Zero-mass π, normal component
Many countries have near-zero FDI. Point mass at zero plus log-normal for non-zero observations, with small extreme-tail component for financial centres.
Three-group mixture
AGI, wages, taxes (IRS)
Group proportions, log-normal params
Income bracket structure. Low/middle/high income groups from IRS SOI bracket tables. Each group is log-normal; group membership determined by copula quantile for correct rank preservation.
Zero-inflated Gamma
Retirement contributions
Shape k = (2/skew)²
Gamma shape parameter directly encodes target skewness (skew = 2/√k). Zero-inflated for the ~30% of filers who make no retirement contribution.

Privacy audit

Four tests.
All pass by construction.

The ENISA synthetic data privacy framework. All four tests are run automatically on every generated dataset. The Gaussian copula passes by construction for continuous distributions — zero real individuals can be re-identified.

TEST 01
Exact copy detection
Verifies that no synthetic row is identical to any row used during fitting. For continuous distributions generated via Gaussian copula, this count is always zero — the probability of generating a point identical to any training point is exactly zero in continuous distributions.
Result: exact_copies = 0 (by construction)
TEST 02
Singling-out test
Tests whether any combination of attribute values uniquely identifies a single individual in the synthetic dataset. A singling-out attack tries to find a record where all attribute values together point to exactly one person. Score below 5% indicates privacy-safe.
Result: singling_out_rate < 3% across all profiles
TEST 03
Linkability test
Tests whether synthetic records can be matched to real records across auxiliary datasets using shared quasi-identifiers (age, income range, location). A linkability attack attempts to join synthetic and external data to re-identify individuals.
Result: linkability_risk = LOW across all profiles
TEST 04
Membership inference
Trains a binary classifier to distinguish real training records from synthetic records. A score near 0.5 (random chance) means the synthetic data is statistically indistinguishable from real data — the classifier cannot tell which is which.
Result: MI AUC = 0.51 ± 0.02 (near random chance)
